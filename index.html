Lumo.CSS

Documentation page

Introduction

Welcome!

Welcome to Lumo.CSS, which is low-class CSS and JavaScript framework for web pages. It includes nearly 30 pre-styled components ready to use in your website with ability to customize some of the elements of the framework.

The main idea behind this framework is to keep it simple and intuitive so in 90% of the cases it uses semantic HTML elements with aria- attributes with just a little of the CSS classes used in it (less than 15).


Quick start

There are 2 ways to get started with Lumo.CSS:


1st way: Download manually and link files in website
  1. First download lumo.min.css and lumo.min.js.
  2. Then link it in your .html file: 
    <link rel="stylesheet" href="lumo.min.css">
<script src="lumo.min.js"></script>
2nd way: Use jsDelivr CDN in your website
  1. Create website html file
  2. Include CDN links in the <head> section: 
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/y4ch0/Lumo.CSS/1.0.1/dist/lumo.min.css">
<script src="https://cdn.jsdelivr.net/gh/y4ch0/Lumo.CSS/1.0.1/dist/lumo.min.js"></script>

Accessibility

Lumo.CSS includes features for accessibility, such as respecting prefers-reduced-motion. You can also manually disable animations by adding data-reduce-motion="true" to the <body> tag. 

<body data-reduce-motion="true">
  <!-- All animations will be disabled -->
</body>

Dialog (Modal)

Displays content in a modal dialog over the page. Trigger with data-target="modal-id" on a button. Close with data-close on a button or by clicking the backdrop (unless data-locked="true" is set on the <dialog>). You can control the modal's width by adding data-width="s", data-width="l", or data-width="xl" to the inner <section> element.

Example

Dialog title

Lorem ipsum dolor sit amet consectetur adipisicing elit. Veniam qui mollitia itaque et temporibus magnam hic id, optio earum cumque!

Dialog title (small)

This modal has data-width="s".

Cookie Settings

This modal has data-locked="true" and cannot be closed by clicking the background.
Code
<!-- Button to open the modal -->
<button data-target="testing-dialog">Launch modal demo</button>

<!-- The modal dialog -->
<dialog id="testing-dialog">
    <section>
        <header>
            <h4>Dialog title</h4>
            <button class="close" data-close></button>
        </header>
        <p>Modal content...</p>
        <footer>
            <button class="secondary" data-close>Cancel</button>
            <button class="primary">Confirm</button>
        </footer>
    </section>
</dialog>

<!-- Small modal -->
<button data-target="testing-dialog-small">Launch 'small' modal</button>
<dialog id="testing-dialog-small">
    <section data-width="s">
        <header>
            <h4>Dialog title (small)</h4>
            <button class="close" data-close></button>
        </header>
        <p>This modal has <code>data-width="s"</code>.</p>
    </section>
</dialog>

<!-- Locked modal -->
<button data-target="cookie-modal">Launch locked modal</button>
<dialog id="cookie-modal" data-locked="true">
    <section>
        <h2>Cookie Settings</h2>
        <p>This modal is locked.</p>
        <footer>
            <button data-close>Accept</button>
        </footer>
    </section>
</dialog>

Fieldset

Used to group several controls within a web form. Can be stacked (default) or grouped horizontally with role="group". Use .grid for a responsive 2-column layout.

Example
Code

    <label>E-mail:<input type="email" name="email"></label>
</fieldset>

<!-- Grid fieldset -->
<fieldset class="grid">
    <label>Given name:<input type="text" name="name"></label>
    <label>E-mail:<input type="email" name="email"></label>
</fieldset>

<!-- Grouped fieldset -->
<fieldset role="group">
    <select style="width:fit-content">
        <option>+1</option>
    </select>
    <input type="tel" name="tel" placeholder="Phone number">
    <button type="submit">Send</button>
</fieldset>
